# 小团快跑模拟规则文档

本文档描述当前程序实际实现的比赛规则，只记录游戏逻辑和判定顺序，不包含多线程、CUDA、编译等技术实现内容。

## 1. 基本对象

### 1.1 普通团子

- 普通团子的编号范围是 `0..runnerCount-1`。
- 当前工程默认有 6 个普通团子：
  - `0`: 陆·赫斯团子
  - `1`: 西格莉卡团子
  - `2`: 达妮娅团子
  - `3`: 维雪团子
  - `4`: 卡提希娅团子
  - `5`: 菲比团子
- 程序上限是 18 个普通团子。

### 1.2 布大王团子

- 布大王不是普通参赛团子，不会成为胜者。
- 启用布大王时，它的实体编号是 `runnerCount`。
- 布大王有独立行动规则，详见第 8 节。

### 1.3 赛道

- 赛道是一个环形数组。
- 数组下标是程序内部位置，下标从 `0` 开始。
- 普通团子前进方向是数组下标递增方向。
- 位置移动会自动环绕，例如从最后一格继续前进会回到第 `0` 格。
- 程序最多支持 256 个赛道格。

### 1.4 格子类型

当前实现有 5 种格子：

| 编号 | 类型 | 规则 |
| --- | --- | --- |
| `0` | 常规格 | 无效果 |
| `1` | 推进装置 | 移动终点为该格时，继续沿当前行动方向移动 `amount` 格；地图文件中默认是 1 格 |
| `2` | 阻遏装置 | 移动终点为该格时，向当前行动方向的反方向移动 `amount` 格；地图文件中默认是 1 格 |
| `3` | 时空裂隙 | 移动终点为该格时，随机重排该格堆叠顺序；若布大王在该格，布大王仍保持在底部 |
| `4` | 终点 | 作为胜利判定目标；落在终点格本身不会触发额外格子效果 |

## 2. 开局规则

### 2.1 行动顺序

- 每一场比赛开局时生成一组普通团子的行动顺序。
- 当前默认配置会随机打乱普通团子顺序。
- 行动顺序在一场比赛内固定不变，每个回合都按同一个顺序行动。
- 若关闭随机行动顺序，则行动顺序为 `0,1,2,...`。

### 2.2 初始堆叠

- 所有普通团子从 `startIndex` 出发。
- 当前默认配置把行动顺序解释为“从上到下”。
- 因此 `actionOrder[0]` 是最上层团子，也是普通团子里第一个行动的团子。
- `actionOrder[runnerCount-1]` 是最下层普通团子。
- 程序内部堆叠深度越大，表示越靠上。

### 2.3 地图文件载入后的起点和终点

- 使用 `--map` 载入地图文件时：
  - 地图数组最后一个值必须是 `4`。
  - 终点被设为地图数组最后一格。
  - 起点默认被设为数组第 `0` 格。
- 可以通过 `--start-index` 覆盖起点。
- `--start-index` 使用程序内部的 0-based 下标。

## 3. 回合流程

每一场比赛最多执行 10000 回合。若 10000 回合内没有胜者，程序报错。

每个回合按以下步骤执行：

1. 清空所有普通团子的“本回合移动惩罚”。
2. 若启用布大王，且当前回合数大于等于 `budaStartRound`，布大王先行动一次。
3. 普通团子按本场比赛开局生成的行动顺序依次行动。
4. 若启用布大王，且当前回合数大于等于 `budaStartRound`，回合结束时执行布大王回合结束检查。
5. 若任意步骤产生胜者，比赛立即结束，不再继续后续行动。

默认配置中：

- 普通团子骰子范围是 `1..3`。
- 布大王从第 3 回合开始行动。
- 布大王骰子范围是 `1..6`。

## 4. 普通团子行动流程

一个普通团子的行动按以下顺序结算：

1. 若该团子有 `skipTurn` 标记，则清除该标记并跳过本次行动。
2. 投普通骰子，得到 `roll`。
3. 初始移动步数 `move = roll`。
4. 结算该团子的 `BeforeRoll` 效果。
5. 结算该团子的 `AfterRoll` 效果。
6. 若该团子的“末位爆发”已激活，则按保存的概率尝试给 `move` 增加对应步数。
7. 若该团子本回合有移动惩罚，则执行 `move = max(1, move - penalty)`。
8. 结算该团子的 `BeforeMove` 效果。
9. 按 `move` 移动该团子以及它上方的所有团子。
10. 若移动已经产生胜者，比赛结束。
11. 结算落点格子效果，包括推进、阻遏和时空裂隙。
12. 若格子效果产生胜者，比赛结束。
13. 结算该团子的 `AfterMove` 效果。
14. 结算该团子的 `OnLand` 效果。
15. 结算该团子的 `EndTurn` 效果。
16. 将该团子的 `lastRoll` 更新为本次 `roll`。

若某个技能效果在前面的步骤中直接产生胜者，后面的步骤不会继续执行。

## 5. 移动和堆叠规则

### 5.1 移动组

- 当一个实体移动时，程序会取出该实体以及它上方的所有实体，形成一个移动组。
- 移动组保持原有相对堆叠顺序。
- 被带着移动的团子也会改变位置。

### 5.2 移动距离和位置

- 目标位置为 `wrap(from + steps)`。
- `steps > 0` 表示沿普通团子前进方向移动。
- `steps < 0` 表示反方向移动。
- 环形赛道使用取模环绕。

### 5.3 累计前进距离

- 移动组里的普通团子会更新累计前进距离：
  - `progress = max(0, progress + steps)`
- 因此后退会减少累计前进距离，但不会低于 0。
- 布大王不参与普通团子的累计前进距离。

### 5.4 落到已有堆叠上

- 普通团子移动组落到已有堆叠格时，移动组会叠到该格最上方。
- 若移动组的底部是布大王，则该移动组会插入到目标格底部。
- 这保证布大王始终位于所在格的底部。

### 5.5 排名

排名用于西格莉卡标记、末位判断、布大王回合结束判断等。

排名规则如下：

1. 累计前进距离 `progress` 更大者排名更高。
2. 若累计前进距离相同，堆叠深度更大者排名更高，也就是越靠上排名越高。

## 6. 胜利判定

### 6.1 基本胜利条件

- 普通团子沿前进方向移动时，只要移动路径抵达或越过终点，就记为经过终点。
- 不要求刚好停在终点。
- 只有 `steps > 0` 的移动会进行终点经过判定。
- 后退不会获胜。
- 布大王移动不会获胜。

### 6.2 多圈判定

- 程序保存每个普通团子的 `finishCrossings`。
- 每一次前进移动会检查从第 1 步到第 `steps` 步是否经过终点。
- 若一次移动跨过终点多次，会增加多次经过次数。
- 当普通团子的 `finishCrossings >= lapsToWin` 时，该团子满足胜利条件。
- 当前默认 `lapsToWin = 1`。

### 6.3 堆叠同时过终点

- 如果一个移动组里多个普通团子同时满足胜利条件，则移动组中最上层的满足条件团子获胜。
- 产生胜者时，CPU 轨迹记录会把该移动组放到终点格。

## 7. 格子效果结算

普通移动结束后，会根据当前行动实体的落点结算格子效果。

### 7.1 常规格

- 无效果。
- 格子效果链结束。

### 7.2 终点格

- 终点格本身无额外落地效果。
- 胜负已经在移动路径经过终点时判定。
- 格子效果链结束。

### 7.3 推进装置

- 推进装置基础移动为沿当前行动方向移动 `amount` 格。
- 地图文件中的推进装置 `amount` 为 1。
- 对普通团子而言，推进是向数组下标递增方向移动。
- 对布大王而言，推进是沿布大王当前行动方向移动，也就是数组下标递减方向。
- 若当前行动实体是普通团子，会先结算该团子的 `OnAdvanceTile` 效果，允许修改本次推进装置的移动步数。
- 然后移动当前行动实体以及它上方的移动组。
- 移动后继续检查新的落点格子效果。

### 7.4 阻遏装置

- 阻遏装置基础移动为向当前行动方向的反方向移动 `amount` 格。
- 地图文件中的阻遏装置 `amount` 为 1。
- 对普通团子而言，阻遏是向数组下标递减方向移动。
- 对布大王而言，阻遏是向数组下标递增方向移动。
- 若当前行动实体是普通团子，会先结算该团子的 `OnDelayTile` 效果，允许修改本次阻遏装置的移动步数。
- 然后移动当前行动实体以及它上方的移动组。
- 移动后继续检查新的落点格子效果。

### 7.5 时空裂隙

- 时空裂隙会随机重排当前格的堆叠顺序。
- 若布大王位于该格底部，布大王保持底部不参与重排。
- 时空裂隙结算后，格子效果链结束。

### 7.6 格子效果链上限

- 推进和阻遏可能连续触发新的格子效果。
- 程序允许的格子效果链最大长度是 `trackLength * 2`。
- 若超过该上限仍未终止，程序报错。

## 8. 布大王规则

### 8.1 初始状态

- 启用布大王时，布大王初始位于终点格。
- 布大王插入终点格堆叠底部。

### 8.2 行动时机

- 当 `round >= budaStartRound` 时，布大王在每回合普通团子行动前先行动。
- 默认 `budaStartRound = 3`。

### 8.3 行动方式

- 布大王投 `budaDiceMin..budaDiceMax` 的骰子。
- 默认是 `1..6`。
- 布大王移动步数为负数，即朝普通团子前进方向的反方向移动。
- 布大王移动时也会带着自己上方的实体一起移动。
- 布大王可以触发推进、阻遏、时空裂隙。
- 布大王触发推进或阻遏时，不会结算普通团子的装置触发技能。
- 布大王不能通过经过终点获胜。

### 8.4 相遇记录

- 每当实体移动、重排或传送后，程序会检查哪些普通团子与布大王处于同一格。
- 与布大王处于同一格的普通团子会永久获得 `metBudaKing` 标记。
- 该标记本场比赛内不会清除。

### 8.5 回合结束传送

- 在布大王开始行动后的每个回合结束时，程序找出当前最后一名普通团子。
- 如果最后一名普通团子与布大王不在同一格，布大王会传送回终点格底部。
- 如果最后一名普通团子与布大王在同一格，布大王不传送。

## 9. 技能系统

每个技能由以下字段组成：

- `trigger`: 触发时机。
- `op`: 效果类型。
- `chancePermille`: 千分概率，`1000` 表示必定触发，`0` 表示不触发。
- `a`: 参数 1。
- `b`: 参数 2。

### 9.1 概率判定

- `chancePermille >= 1000` 时必定命中。
- `chancePermille <= 0` 时必定不命中。
- 其他情况随机取 `1..1000`，若结果小于等于 `chancePermille`，则命中。

### 9.2 触发时机

当前枚举了以下触发时机：

| 触发时机 | 当前结算位置 |
| --- | --- |
| `BeforeRoll` | 普通团子投骰后，`AfterRoll` 前 |
| `AfterRoll` | 投骰后，末位爆发和惩罚前 |
| `BeforeMove` | 普通团子正式移动前 |
| `AfterMove` | 普通团子移动和落点格子效果后 |
| `OnLand` | `AfterMove` 后的额外通用落地触发 |
| `OnAdvanceTile` | 当前行动普通团子落到推进装置时 |
| `OnDelayTile` | 当前行动普通团子落到阻遏装置时 |
| `EndTurn` | 普通团子行动末尾 |
| `StartRound` | 已定义但当前行动流程没有调用 |

### 9.3 效果类型

| 效果类型 | 规则 |
| --- | --- |
| `None` | 无效果 |
| `AddFlatMove` | 若概率命中，`move += a` |
| `AddRandomMove` | 若概率命中，`move += random(min(a,b)..max(a,b))` |
| `AddIfSameAsPreviousRoll` | 若 `lastRoll > 0` 且本次 `roll == lastRoll`，`move += a` |
| `MarkAheadPenalty` | 标记排名紧邻自身且更高的至多 `b` 个普通团子，本回合移动惩罚为至少 `a` |
| `MoveSelf` | 当前团子立刻移动 `a` 格，可触发胜利判定，但不会额外结算落点格子效果 |
| `MoveLeader` | 当前第一名立刻移动 `a` 格，可触发胜利判定，但不会额外结算落点格子效果 |
| `MoveLast` | 当前最后一名立刻移动 `a` 格，可触发胜利判定，但不会额外结算落点格子效果 |
| `SkipSelfTurn` | 给当前团子设置跳过下次行动标记 |
| `ActivateLastPlaceSurge` | 若当前团子尚未消耗该效果且此时处于最后一名，则激活后续末位爆发 |
| `EnableAfterMeetingBuda` | 若当前团子已经遇到过布大王，`move += a` |

### 9.4 标记和惩罚

- `MarkAheadPenalty` 使用当前排名。
- 它只会向排名高于自己的方向查找。
- 最多标记 `b` 个团子。
- 同一回合内若同一团子被多次标记，保留较大的惩罚值。
- 被惩罚团子移动时执行 `move = max(1, move - penalty)`。
- 惩罚在每个回合开始时清空。
- 若目标团子本回合已经行动过，该惩罚不会影响已经结束的行动。

### 9.5 末位爆发

- `ActivateLastPlaceSurge` 不立即增加当前移动步数。
- 它只是在满足条件时激活一个持续状态。
- 激活条件是：该团子尚未消耗过该效果，且触发时该团子处于最后一名。
- 激活后保存概率和增加步数。
- 从之后每次该团子行动开始，都会在惩罚结算前按保存概率尝试增加步数。
- 每场比赛最多激活一次。

## 10. 当前默认团子技能

### 10.1 陆·赫斯团子

- 推进装置触发：`OnAdvanceTile + AddFlatMove(1000, +3)`。
- 阻遏装置触发：`OnDelayTile + AddFlatMove(1000, -1)`。
- 因此：
  - 落到推进装置时，基础推进 `+1` 会变成总计 `+4`。
  - 落到阻遏装置时，基础阻遏 `-1` 会变成总计 `-2`。
- 只有陆·赫斯本人是当前行动实体时才会触发。
- 如果陆·赫斯只是被别的团子带着落到装置上，不会触发该技能。

### 10.2 西格莉卡团子

- 投骰后触发：`AfterRoll + MarkAheadPenalty(1000, amount=1, maxTargets=2)`。
- 标记排名紧邻自己且高于自己的至多两个普通团子。
- 被标记团子本回合移动步数减少 1，最低为 1。

### 10.3 达妮娅团子

- 投骰后触发：`AfterRoll + AddIfSameAsPreviousRoll(1000, +2)`。
- 若本次投骰点数与自己上一次完成行动时的点数相同，则本次移动步数加 2。
- 第一次行动时 `lastRoll = 0`，不会触发。

### 10.4 维雪团子

- 移动前触发：`BeforeMove + EnableAfterMeetingBuda(1000, +1)`。
- 只要本场比赛中曾经与布大王位于同一格，则之后每次移动前步数加 1。

### 10.5 卡提希娅团子

- 移动和格子效果结束后触发：`AfterMove + ActivateLastPlaceSurge(600, +2)`。
- 若此时卡提希娅处于最后一名，且本场还未消耗过该效果，则激活末位爆发。
- 激活之后，卡提希娅每次行动有 60% 概率额外前进 2 格。
- 该激活每场比赛最多发生一次。

### 10.6 菲比团子

- 投骰后触发：`AfterRoll + AddFlatMove(500, +1)`。
- 每次行动有 50% 概率额外前进 1 格。

## 11. 随机事件

当前规则中会使用随机数的地方包括：

- 每场比赛开局行动顺序随机打乱。
- 普通团子投骰。
- 布大王投骰。
- 概率技能判定。
- `AddRandomMove` 的随机步数。
- 时空裂隙重排堆叠。
- 轨迹抽样是否记录。

默认运行时，主程序会自动生成随机种子。若通过 `--seed` 指定种子，则同样参数会得到可复现结果。
